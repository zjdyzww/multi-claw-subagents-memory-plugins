# Memory System Specification

> 多智能体记忆系统规范 v2.0

## 1. 系统概述

本记忆系统为多智能体环境提供分层、统一、可追溯的记忆管理能力。

### 1.1 设计原则

| 原则 | 说明 |
|------|------|
| **分层隔离** | L1-L4 四层架构，每层职责明确 |
| **按需共享** | 私有记忆隔离，公共记忆按权限共享 |
| **版本可控** | Git 版本控制，支持回溯和审计 |
| **性能优先** | 分层缓存，检索延迟 < 100ms |

## 2. 记忆层级

### L1: 核心协议层

**职责**: 存储核心协议和身份定义

**内容**:
- 智能体身份定义 (L1_CORE.md)
- 排他性规则
- 通信协议 (MCP)
- 访问控制协议

**容量**: ~10KB/智能体

### L2: 业务规则层

**职责**: 存储业务规则和决策

**内容**:
- 任务分配策略
- 协作模式规范
- 权限矩阵
- 项目决策记录

**容量**: ~50KB/智能体

### L3: 配置层

**职责**: 存储运行环境配置

**内容**:
- 软硬件环境信息
- API 端点配置
- 依赖版本信息
- 共享环境变量

**容量**: ~20KB/智能体

### L4: 索引层

**职责**: 提供快速检索入口

**内容**:
- 记忆索引文件
- 版本历史
- 搜索元数据

**容量**: ~5KB/智能体

## 4. 同步机制

### 同步类型

| 类型 | 触发条件 | 范围 | 冲突解决 |
|------|----------|------|----------|
| 定时同步 | 10:00, 22:00 | 全量 | LWW |
| 按需同步 | API 调用 | 增量 | 审核 |
| 实时同步 | 重要变更 | 差异 | 广播 |

### 冲突解决策略

1. **LWW (Last Write Wins)**: 按时间戳
2. **Priority**: 按智能体优先级
3. **Review**: Hermes 审核
4. **Manual**: 人工介入

## 5. 访问控制

### 权限级别

| 级别 | 描述 | 适用场景 |
|------|------|----------|
| PRIVATE | 仅创建者可访问 | 私有记忆 |
| AGENT_LOCAL | 同类型智能体可读 | 排他性规则 |
| SHARED | 所有智能体可读 | 公共知识 |
| SHARED_WRITE | 所有智能体可读写 | 主仓核心层 |

### 权限矩阵

| 仓库 | OpenClaw | Hermes | Claude-Code | OpenCode |
|------|----------|--------|-------------|----------|
| 主仓 L1-L2 | RW | RW | R | R |
| 主仓 L3-L4 | RW | RW | RW | RW |
| 业务仓 | R | RW | - | R |
| 代码仓 | R | R | RW | RW |
| 私有仓 | - | - | - | RW |

## 6. 性能指标

| 指标 | 目标 | 测量方法 |
|------|------|----------|
| 检索延迟 | < 100ms | P50 响应时间 |
| 同步成功率 | 99% | 成功/总数 |
| 记忆命中率 | 85% | 命中/查询 |
| 存储效率 | < 52KB/智能体 | 平均文件大小 |

## 7. 安全要求

- TLS 1.3 传输加密
- AES-256 存储加密
- 完整审计日志
- GDPR Article 17 合规
