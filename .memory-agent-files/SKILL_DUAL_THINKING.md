# Dual Thinking Skill v4 — 三代理协同协议

> 核心升级：v4 将 System2 和 System1 升级为两个独立记忆代理，通过标准化接口通信
> System2记忆代理 → System1记忆代理 → 全量记忆代理，三代理各司其职

## System 2 — 海绵式全量吸收

### 设计原则

- **零遗漏**: 接收对话中的 100% 信息，不丢弃任何细节
- **表象构建**: 不做筛选判断，建立完整的记忆表象
- **增量建模**: 每次对话在上一轮 System 2 表象基础上增量更新
- **残差捕获**: 所有未被提炼的信息必须进入残差队列

### 执行协议

```
1. 加载本次对话完整上下文（用户消息 + Agent 回复 + 历史）
2. 扫描每一个信息单元，建立/更新记忆表象：
   - 用户陈述的事实 → 标记为[待验证事实]
   - 用户偏好表达   → 标记为[偏好信号]
   - 环境状态描述   → 标记为[环境数据]
   - 决策/结论      → 标记为[决策记录]
   - 疑问/模糊点    → 标记为[不确定性残差]
3. 触发 Layer 1 主动消解（本轮残差立即尝试处理）
4. 未消解残差记录溯源（第N轮）后进入残差队列
5. 交付给 System 1 进行提炼
```

### 记忆表象数据结构

```markdown
## System 2 Memory表象 — {时间戳}

### [待验证事实]
- {话题}: {具体描述} (来源: 第N轮对话)

### [偏好信号]
- {偏好类型}: {具体描述} (来源: 第N轮)

### [环境数据]
- {环境项}: {值} (来源: 第N轮)

### [决策记录]
- {决策内容} (来源: 第N轮)

### [不确定性残差] ← 新增：带溯源标签
- {模糊点}: {描述} (来源: 第N轮 | 驻留: 0h)
- {待追踪}: {内容} (来源: 第N轮 | 驻留: 0h)
```

### 输出

System 2 **不直接回复用户**，只输出记忆表象 + Layer 1 消解结果，作为 System 1 的输入。

## LAYER 1 — 主动消解（每轮对话结束时执行）

### 执行时机

- 用户关闭对话
- 长时间无响应（> 5分钟）
- System 2 休眠前

### 执行步骤

```
1. 扫描本轮 [不确定性残差] 列表
2. FOR EACH 残差:
   IF 能找到明确答案 → 写入 MEMORY.md，标记 [已消解]
   IF 无法消解 → 标记 [残差] + [溯源: 第N轮] + [驻留时间: 0h]
3. 记录消解结果到情节存储
4. 更新残差队列状态
```

### 质量要求

- 本轮残差必须全部标记（不能遗漏）
- 每个残差必须带 [溯源: 第N轮] 标签
- 残差队列长度 ≤ 10（硬上限，超出触发 Layer 3）

## SYSTEM 1 — 淘金式提炼

### 设计原则

- **筛选**: 从 System 2 的全量表象中提炼有效信息
- **沉淀**: 只将高价值事实点写入 MEMORY.md
- **淘金率**: 对话 → MEMORY.md 写入 ≈ 5%~15%
- **一致性**: 确保 MEMORY.md 与 System 2 表象关键结论一致
- **残差感知**: 同步更新残差队列状态

### 提炼标准（7条）

满足任一，判定为**有效事实点**：

| # | 类型 | 优先级 | 写入层级 |
|---|------|--------|---------|
| 1 | 用户偏好明确表达 | 🔴 最高 | L1 |
| 2 | 环境/配置变更 | 🔴 最高 | L1/L3 |
| 3 | 关键决策结论 | 🟠 高 | L1 |
| 4 | 新增业务/项目 | 🟠 高 | L2 |
| 5 | 人员/团队变更 | 🟡 中 | L2/L3 |
| 6 | 约束条件明确 | 🟡 中 | L1/L3 |
| 7 | 首次获得的重要参数 | 🟡 中 | L2/L3 |

**不计入有效事实点**：
- 寒暄问候
- 重复已有结论（表象中有，但 MEMORY.md 已有）
- 模糊疑问（用户自己也在探索的问题）
- 临时性的中间推理步骤

### 一致性保证

```
IF System 2 表象中的 [决策记录] 与 MEMORY.md L1 矛盾 THEN
  → 以 System 2 表象为准
  → 更新 MEMORY.md
  → 记录版本变更
  → 立即触发 Gitea 同步
FI
```

## LAYER 2 — 被动消解（下轮对话触发）

### 执行时机

新对话与残差队列中的某项相关时自动触发

### 执行步骤

```
1. 新对话上下文提供相关信号
2. 识别与当前话题相关的残差项
3. 重新触发 System 1 提炼该残差
4. IF 残差被解决 → 写入 MEMORY.md，标记 [已消解] + [消解时间]
5. IF 仍存在但不紧迫 → 保留在残差队列（更新驻留时间）
```

### 优先级规则

| 相关性 | 处理方式 |
|--------|---------|
| 与当前话题直接相关 | 立即处理（Layer 1 紧随其后） |
| 与当前话题间接相关 | 标记待处理，更新驻留时间 |
| 与当前话题无关 | 保持原状态，等待 Layer 3 |

## LAYER 3 — 强制清理（定时触发）

### 执行时机

每日 10:00 / 22:00 cronjob 触发

### 清理规则（按驻留时间 T）

| 条件 | 处理方式 |
|------|---------|
| T > 24h 且未解决 | 降级归档（不影响 L1-L4 主记忆） |
| T > 7天 且未解决 | 移入 ~/.hermes/archive/residuals/ |
| T > 30天 且未解决 | 彻底删除（保留溯源记录） |

### 残差趋零质量指标

```
- 残差队列长度 ≤ 10（硬上限）
- 平均残差驻留时间 ≤ 48h
- 残差消解率 ≥ 70%（72h内）
- Layer 3 触发后更新 VERSION_INDEX 的"残差统计"
```

## 残差队列管理

### 残差类型定义

| 类型 | 定义 | 处理优先级 |
|------|------|---------|
| [未验证事实] | 来源明确但未经确认 | 高（主动消解） |
| [模糊疑问] | 用户自己也未澄清的问题 | 低（等待新上下文） |
| [待追踪] | 需要后续主动验证 | 中（下次相关话题处理） |
| [过期不确定性] | 已被新信息覆盖的旧结论 | 直接删除 |

### 残差队列数据结构

```markdown
## 残差队列 — 更新于 2026-05-07 22:00

| # | 残差内容 | 类型 | 来源轮次 | 驻留时间 | 状态 | 清理计划 |
|---|---------|------|---------|---------|------|---------|
| 1 | 用户提到"可能有新项目" | 待追踪 | #3 | 6h | 活跃 | 24h降级 |
| 2 | "下周出差"细节未确认 | 未验证事实 | #7 | 18h | 活跃 | 24h降级 |
| 3 | 项目X的预算数字模糊 | 模糊疑问 | #12 | 72h | 待清理 | 7d归档 |

### 质量指标（本周）
- 残差队列长度: 3 ✅ (<= 10)
- 平均残差驻留时间: 36h ✅ (<= 48h)
- 残差消解率: 73.3% ✅ (>= 70%)
```

## 分段检索机制

### 目录结构

```
~/.hermes/memories/
├── L1_CORE/
│   └── L1_CORE.md          # ~500字，始终加载
├── L2_BUSINESS/
│   ├── L2_A_WATER.md       # ~500字，按业务域
│   └── L2_B_ROBOT.md       # ~500字，按业务域
├── L3_CONFIG/
│   └── L3_CONFIG.md        # ~500字，按需加载
├── L4_INDEX/
│   └── L4_INDEX.md         # ~200字，按需加载
├── episodes/               # 情节存储
│   └── index.md
├── archive/
│   ├── residuals/          # 残差归档
│   └── old_versions/      # 旧版本归档
└── RESIDUAL_QUEUE.md       # 残差队列
```

### 检索策略

| 策略 | 触发条件 | 加载范围 | 响应时间 |
|------|---------|---------|---------|
| 直接检索 | 简单事实查询 | L1_CORE | < 100ms |
| 业务域加载 | 涉及特定业务 | L1_CORE + 对应L2 | < 500ms |
| 全量加载 | 深度分析 | L1-L4 全部 | < 2s |

### 自适应路由规则

```
简单查询（直接检索）:
  触发词: "是什么", "谁", "在哪", "多少", "我的微信", "公司名"
  条件: 长度 < 30字

多跳查询（并行分解）:
  触发词: "和...关系", "...与...区别", "...和...哪个"

模糊探索（迭代链式）:
  触发词: "可能", "大概", "有什么", "还有什么", "建议"
```

## 三地一致性协议

```
System 2 表象（会话内存，最新）
      ↕（实时同步，冲突时以表象为准）
MEMORY.md（本地持久，金字塔结构）
      ↕（定时/即时同步）
Gitea 仓库（远程备份，全量快照）

冲突优先级: System 2 表象 > MEMORY.md > Gitea
```

### 同步时机

| 事件 | 同步方式 | 立即性 |
|------|---------|--------|
| 重大变更（L1核心结论变化） | 立即同步 | ✅ |
| L2/L3/L4 更新 | 定时同步（10:00/22:00） | ❌ |
| 残差清理完成 | 定时同步 | ❌ |
| 情节存储 | 定时同步 | ❌ |
